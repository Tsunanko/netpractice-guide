# 03. CIDR 早見表と計算

## このページは何？

`/24` とか `/25` という書き方（**CIDR 表記** = サイダー表記）の意味と、
**1 つの町に何人住めるか** を 3 秒で計算するコツのページです。

---

## このページで学ぶこと

- CIDR 表記は「マスクで 1 が連続している個数」
- `/24` → 住人 254 人、`/30` → 住人 2 人…の感覚を掴む
- ホスト数 = **2^(32 − マスク桁数) − 2**
- /25 以下で町を分割する考え方

---

## 🧩 CIDR って何？

!!! info "正式名称"
    CIDR = **Classless Inter-Domain Routing**（クラスレス・インタードメイン・ルーティング）。
    「/24」のような表記を **CIDR 表記** と呼ぶ。読み方は「サイダー」。

### 書き方は超シンプル

**マスクの 1 が何個並んでいるかを数字で書くだけ**。

| マスク（10 進） | 2 進（1 の個数） | CIDR 表記 |
|:---|:---|:-:|
| `255.255.255.0` | 1 が 24 個 | **/24** |
| `255.255.255.128` | 1 が 25 個 | **/25** |
| `255.255.255.192` | 1 が 26 個 | **/26** |
| `255.255.255.240` | 1 が 28 個 | **/28** |
| `255.0.0.0` | 1 が 8 個 | **/8** |

---

## 📊 早見表（これを暗記すると早い）

!!! tip "NetPractice 頻出のマスク"
    Level 1〜10 で出てくるのはほぼ **/8, /16, /18, /24, /25, /26, /27, /28, /29, /30** です。

| CIDR | マスク | 1 町のアドレス数 | 住人数（ホスト数） | よくある用途 |
|:-:|:---|:-:|:-:|:---|
| /8  | `255.0.0.0`         | 16,777,216 | 16,777,214 | 大企業全体 |
| /16 | `255.255.0.0`       | 65,536     | 65,534     | 中企業全体 |
| /18 | `255.255.192.0`     | 16,384     | 16,382     | 大きめ LAN |
| /24 | `255.255.255.0`     | 256        | 254        | **家庭・小オフィス** |
| /25 | `255.255.255.128`   | 128        | 126        | 小 LAN |
| /26 | `255.255.255.192`   | 64         | 62         | 小 LAN |
| /27 | `255.255.255.224`   | 32         | 30         | 小 LAN |
| /28 | `255.255.255.240`   | 16         | 14         | 部署 |
| /29 | `255.255.255.248`   | 8          | 6          | 小グループ |
| /30 | `255.255.255.252`   | 4          | **2**      | **ルータ間専用** |
| /31 | `255.255.255.254`   | 2          | 2*         | 点対点リンク |
| /32 | `255.255.255.255`   | 1          | 1*         | 単体ホスト |

*/31, /32 は特殊（RFC 3021）で、NetPractice ではあまり出ない。

---

## 🧮 計算公式

!!! info "覚えるのはこれだけ"
    **1 町のアドレス数** = 2 ^ (32 − CIDR)
    **住人数**         = アドレス数 − 2 （ネットワーク + ブロードキャストを引く）

### 例: /26 なら？

```
アドレス数 = 2^(32-26) = 2^6 = 64
住人数     = 64 - 2 = 62
```

### 例: /28 なら？

```
アドレス数 = 2^(32-28) = 2^4 = 16
住人数     = 16 - 2 = 14
```

---

## 🗂️ /24 を /25 に分けるとどうなる？

大きい町を **半分ずつ** に分けると **/25 が 2 つ** になる。

```
192.168.1.0/24  (256 アドレス)
       ↓ 2 分割
┌─────────────────┬─────────────────┐
192.168.1.0/25     192.168.1.128/25
  .0 〜 .127         .128 〜 .255
  (128 アドレス)     (128 アドレス)
```

各ブロックを拡大:

```
192.168.1.0/25 ブロック
  ├── .0     = ネットワーク（使えない）
  ├── .1〜.126 = 住人 126 人
  └── .127   = ブロードキャスト（使えない）

192.168.1.128/25 ブロック
  ├── .128   = ネットワーク（使えない）
  ├── .129〜.254 = 住人 126 人
  └── .255   = ブロードキャスト（使えない）
```

!!! warning "/25 以下では .255 が使えるブロックがある"
    素朴に「`.255` は使えない」と覚えていると **/25 以下で間違う**。
    正しくは: **各ブロックの最後の番地** が使えない。
    `/25` の場合、`.127` と `.255` の両方が使えない（別ブロックのブロードキャスト）。

## 📏 さらに細かく: /26 で 4 分割

```
192.168.1.0/24 → /26 で 4 分割
┌──────────────┬──────────────┬───────────────┬───────────────┐
.0/26            .64/26          .128/26         .192/26
.0〜.63          .64〜.127        .128〜.191      .192〜.255
62 人           62 人            62 人           62 人
```

**ブロックの区切り目** は **64 の倍数**（= マスク最後の桁の値）。

| CIDR | ブロックサイズ | ブロックの区切り |
|:-:|:-:|:---|
| /24 | 256 | `.0` だけ |
| /25 | 128 | `.0`, `.128` |
| /26 | 64  | `.0`, `.64`, `.128`, `.192` |
| /27 | 32  | `.0`, `.32`, `.64`, ..., `.224` |
| /28 | 16  | `.0`, `.16`, `.32`, ..., `.240` |
| /30 | 4   | `.0`, `.4`, `.8`, ..., `.252` |

## 🎯 「この IP はどのブロックに属する？」を一瞬で求める

NetPractice で一番使うテクニックです。

!!! tip "計算手順"
    1. ブロックサイズ = 256 − マスク最後の桁 の値
    2. IP の最後の桁をブロックサイズで割って **切り捨て** × ブロックサイズ = ブロック先頭

### 例: `192.168.1.222 /27` はどのブロック？

```
マスク /27 = 255.255.255.224
ブロックサイズ = 256 − 224 = 32
222 ÷ 32 = 6.9375 → 切り捨て 6
6 × 32 = 192

→ .192 〜 .223 のブロックに属する
→ 住人範囲: .193 〜 .222
→ ネットワークアドレス: .192
→ ブロードキャスト:     .223
```

### 例: `192.168.1.132 /28` はどのブロック？

```
ブロックサイズ = 256 − 240 = 16
132 ÷ 16 = 8.25 → 切り捨て 8
8 × 16 = 128

→ .128 〜 .143 のブロックに属する
→ 住人範囲: .129 〜 .142
```

## 🔢 /18 みたいな「中途半端な」マスクの扱い

### 例: `150.40.107.254 / 18` の町は？

`/18` は **第 3 オクテット の途中で切る**。

```
150.40.107.254 /18
マスク = 255.255.192.0 (第 3 オクテットが 192)
```

第 3 オクテットのブロックサイズ = 256 − 192 = **64**。

```
107 ÷ 64 = 1.67 → 切り捨て 1
1 × 64 = 64

→ 第 3 オクテットは .64 で始まる
→ 町: 150.40.64.0 / 18
→ 住人範囲: 150.40.64.1 〜 150.40.127.254
```

!!! info "第 3 オクテットの早見表（/18 系）"
    /17 → 128 区切り：`.0, .128`
    /18 → 64 区切り： `.0, .64, .128, .192`
    /19 → 32 区切り： `.0, .32, .64, ..., .224`

---

## 💡 暗算のコツ

!!! tip "256 から引いて考える"
    マスクの最後の桁を見たら、**256 からその値を引く** とブロックサイズになる。

    | マスク最後の桁 | ブロックサイズ |
    |:-:|:-:|
    | 0 | 256 |
    | 128 | 128 |
    | 192 | 64 |
    | 224 | 32 |
    | 240 | 16 |
    | 248 | 8 |
    | 252 | 4 |

---

## ⚠️ よくあるミス

!!! warning "CIDR とマスクの変換を間違える"
    `/27` を `255.255.255.32` と書いてしまうミスが多い。
    正しくは `255.255.255.224`（= 先頭から 1 が 27 個）。

!!! warning "ホスト数の −2 を忘れる"
    「/26 だから 64 人入る」ではなく **62 人**。
    ネットワークとブロードキャストを引き忘れがち。

!!! warning "/30 に 3 人入れる"
    /30 は 4 アドレス中 **2 人しか入らない**。
    ルータ間のリンクで最もよく使う（2 人で十分だから）。

---

## 🎯 まとめ

- CIDR = マスクの 1 の個数を `/24` 形式で書く表記
- 住人数 = 2^(32 − CIDR) − 2
- `/24` = 254 人、`/25` = 126 人 × 2 ブロック、`/30` = 2 人（ルータ間）
- ブロックサイズ = 256 − マスク最後の桁
- 「この IP はどのブロック？」は IP ÷ ブロックサイズ を切り捨てて × ブロックサイズ
